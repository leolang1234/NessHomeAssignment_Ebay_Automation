Import sync playwright — space in import name
Description

The import statement contains a space: sync playwright. Python import names cannot contain spaces, so this is a syntax error that will prevent the module from loading entirely.

Suggested Fix

Remove the space so the name matches the actual module identifier: sync_playwright.
Correct import: from playwright.sync_api import sync_playwright

Wrong Library from selenium import webdriver — wrong framework import
Description

The entire test is written using Playwright's API, yet from selenium import webdriver imports Selenium — a completely different browser automation framework. The webdriver symbol is never used anywhere in the code, so this is dead code. More importantly, it signals a copy-paste mistake that could cause confusion about which framework is in use, and will raise an ImportError if the selenium package is not installed.

Suggested Fix

Remove the line from selenium import webdriver entirely. The project uses Playwright exclusively — the correct import is already present: from playwright.sync_api import sync_playwright.

Resource Leak sync_playwright().start() — no context manager
Description

Calling sync_playwright().start() without a context manager means the Playwright subprocess is never guaranteed to stop. If the test raises an exception before the matching .stop() call, the browser process is left running, leaking system resources.

Suggested Fix

Use the context manager form so cleanup is automatic even on failure:
with sync_playwright() as p:
This ensures stop() is called on exit regardless of exceptions.
4
Logic search_box.fill = (...) — assignment instead of method call
Description

search_box.fill = (...) uses the assignment operator = instead of calling the method. This silently overwrites the fill attribute on the locator object with the argument value instead of typing text into the element. No text is entered and no error is raised.

Suggested Fix

Replace = with parentheses to make it a proper method call:
search_box.fill("playwright testing")

7
Anti-pattern time.sleep(2/3) — blocks Playwright's event loop
Description

time.sleep() is a blocking OS-level sleep that freezes the Python thread. Playwright's synchronous API drives an internal async event loop; sleeping with time.sleep() starves that loop, which can cause timeouts, missed network responses, and flaky tests. It also makes 2/3 evaluate to 0.666… seconds — likely unintentional.

Suggested Fix

Use Playwright's own timer which cooperates with its event loop:
page.wait_for_timeout(2000)  (milliseconds)
Or better yet, replace fixed waits with explicit element-state waits such as page.wait_for_selector() or locator.wait_for() to avoid unnecessary delays.
8
Style Inconsistent indentation throughout the file
Description

The file mixes tab characters and spaces for indentation. Python treats tabs and spaces as different indentation levels. Mixing them causes an IndentationError (Python 3) or silently misaligns logic blocks (Python 2), leading to incorrect control flow or outright parse failures.

Suggested Fix

Standardize on 4 spaces per indent level (PEP 8 recommendation) throughout the entire file. Most editors can convert tabs to spaces automatically. Running pycodestyle or enabling a linter such as flake8 or ruff will catch this class of issue automatically in CI.
